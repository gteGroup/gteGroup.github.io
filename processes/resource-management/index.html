    Resource Management#

    Overview#

    Context#

    GTE personnel ("resources") generally work across several active projects at any one time and so a resource's availability and capacity to complete project work must be assessed from an organizational level (WBS Level 0).

    Budgeted work (ie activities with an assigned budget) is assigned to resources at the project activity level. This is known as "resource assignment". The total work assigned to a resource at any point in time, versus that resource's capacity (generally based on an 8 hour/day), can be considered the "resource loading". The standard GTE Smartsheet schedule template calculates this loading, as a percentage, at the budgeted activity level and reports this up for use on organizational level reports and views.

    As a starting point, the Project Manager will ensure the planned duration for an activity affords sufficient total capacity (duration x hours/day capacity) for the assigned resource to complete the activity in isolation (ie without yet also considering the resource's other work requirements). In simple terms, the total capacity allocated must exceed the activity's budget.

    Example

    For example, a 4 day duration may be established for an activity with a budget of 20hrs. The resource loading (for an 8hr/day resource) would be 20 / (8*4), or 63%.

    The above example simplifies the scenario to a single resource but it should be noted that where the duration is not negotiable, it may be necessary to add additional resources to the activity to increase the total capacity.

    Smartsheet schedules also exist to account for resource assignment that may not be covered by a project schedule:

    • Leave
    • Overheads (may be removed in the future)
    • Non-Working Time (such as schedule person-specific non-working days)
    • Secondments
    • Audits
    • Small Projects - Comms
    • Potential Projects

    Resource Managers will ensure the resources for which they are responsible, are appropriately loaded. This is done using the Resource Views in Smartsheet. It is the Resource Manager's responsibility to ensure that the Resource View for each relevant reflects that resources actual loading, by:

    1. Verifying that all the resource's assignments are visible and accounted for. That may mean a discussion with the relevant Project Manager where it appears a particular project is not showing, or is incorrectly scheduled.
    2. Capturing resource assignments outside of projects - such as leave and secondments - or ensuring with the appropriate administrative person that these are catpured.

    Process#

    As resource capacity management is conducted at the organizational level and resource assignment is at the project activity level, the process inherently involves both project resource management, and organizational resource management. The former is the domain of the Project Manager, and the latter: the Resource Managers.

    Project Resource Management#

    The Project Manager must:

    1. Assign budgets to activities or summary activities in the schedule, such that the overall assigned budget equals the total approved project budget.
    2. Identify activities where the hours/day capacity is not the standard 8 hours ("Non-Standard Working Time") and allocate additional capacity to these Non-Standard Working Time activities.
    3. Establish durations for activities in the schedule by defining start and finish dates, and/or logic that defines start and finish dates.
    4. Assign a resource, or resources, to budgeted activities in the schedule .

    Resource Views#

    Resource Views provide the visibility for the loading of GTE human resources. The views are automatically generated from resource assignments made at the project activity level, and from additional non-project schedules designed to capture auxiliary assignments such as leave and secondments.

    Project Resource Views#

    A Resource View showing all of the resources currently assigned to activities on a project is available to the Project Manager from the Gantt View of the Smartsheet project schedule. To access:

    1. Switch to Gantt View

    2. Select the Resource View icon

      Note: See the Smartsheet documentation here for further details.

    The Project Manager must use the Project Resource View to identify overloaded resources assigned to the project and take action to reduce loading to within capacity. The Project Resource View allows the Project Manager to see allocations on other projects so as to discuss competing requirements with the appropriate Resource Manager and/or other Project Manager.

    Team Resource Views#

    It is not possible to share Resource Views in Smartsheet so these need to be setup for the individual.
