    Dynamic PPM#

    The Dynamic PPM allows changes to be made and applied retrospectively to all existing PPMs. This will alleviate the issue where multiple revisions of the same sheet are applied to Projects.

    One Master PPM sheet will dictate the actions to be shown on to all existing PPMs known as instances. Changes to the Master will reflect and be applied retrospectively to all PPMs.

    Important

    Any changes to any of the PPM sheets, reports or dashboards must follow an agreed procedure and must be properly documented. The steps to achieve each of these requirements are described in this section.

    PPM Provisioning#

    Portal will look for within the 0000 - PM Framework (Restricted) Workspace, for a folder named [TEMPLATE] PPM_Template. A copy of this folder will be made and placed within 1000 - Project Process Management.

    This folder will be rename to the usual Project Name convention - Pxxxx_ClientName-ProjectName. Within this folder the PPM template is to reside with the name [TEMPLATE] PPM_Template.

    Portal will look for this Smartsheet, rename the sheet to xxxx_PPM. It will also look for a field Project ID within the Sheet Summary and update accordingly.

    Adding Rows#

    Rows can be added and applied retrospectively automatically due to the nature of the system. In order to do so, the steps below MUST be followed to ensure the process does not cause unnecessary bugs or faults.

    1. Copy a row you want to base the new row of - consider rules that are attached to the row.
    2. Paste the row into the correct order
    3. Check rules accurately reflect the task
    4. Update the Action column

    5. If a parameter is to be used:

      i. If a new parameter is to be created, add to 1000_PPM_Documentation

      ii. Create a column formula to indicate the required parameter in 1000_PPM_Register

      iii. Update the Parameters column in 1000_PPM_Master
