    Project Register#

    Column Summary#

    Title Brief Description
    # of Disciplines The number of Discipline(s) involved in the project.
    Avaza Client Name Any alternative names used for the Client that are pulled into Avaza.
    Applied Contingency % on Total Proposed Budget Actual contingency % on total proposed budget.
    Awarded & Submitted $ Match Compares the Total Awarded Budget [$] to the Total Proposed Budget [$]. Green 🟢 indicates these match, red 🔴 shows a discrepancy, and grey ◯ will alight when the awarded budget is not yet set.
    Category Projects undertaken by GTE are categorised by: Lump Sum, Rates Based,Audit, Secondment, Onselling, Internal.
    Classification @Potential Classification details can be found in the Classification Overview.
    Classification Classification for a project is determined by its Value Banding and Complexity. All projects fall into one of three classifications: C1, C2 or C3. Details of which can be found in the Classification Overview.
    Client Dropdown box with list of all Clients.
    Client email is NOT in the Master Contacts List If the client email is not in the Master Contacts List Smartsheet, this will be flagged.
    Client Invoice Entity Prioritises the Avaza Client Name over the Client when an optional Avaza Client Name is provided.
    Client PM Client project manager's contact address
    Client Reference Client reference ID for the project.
    Client Site(s) Dropdown box with list of all Client Site(s). For site(s) not listed, choose Site not listed from the dropdown box.
    Client Timesheets Approval Required Indicates whether any of the Avaza timesheets for the project are to be approved by Client's representative (used to automatically set up a respective tag in Avaza to allow filtering).
    Complexity Rating Once all risks and cost contingency are established, the weightings are used to calculate an overall score. The score is translated into a Complexity rating. Please see HERE for more details.
    Contract or PO #(s) Contract or PO #(s) for the project.
    Created By System generated column. Displays email account for the user that created the row entry.
    Created Date System generated column. Displays the date and time for when the row entry was initially created.
    Created in Quarter The quarter (Q1,Q2,Q3 or Q4) of the year in which the project was created in.
    Created in Year System generated column. Displays the year for when the row entry was initially created.
    Creator = Proposal Manager This is a verification checkbox used to determine if the creator of a new potential project via the form is also the proposal manager of the new project. This is to bypass unnessasary steps in the process sending notifications out to the proposal manager.
    Description Brief description of project task.
    Disable Workflows Helper column for smartsheet summary field 'Disable Workflows' functionality. Not to be adjusted manually when wanting to disable workflows. Instead, use the drop-down field in the sheet summary.
    Discipline(s) The discipline(s) involved in the project: Electrical, Controls, Communications, Drafting, Functional Safety, SMP, Project.
    Estimated Project Value vs GTE PM Approval Limit Shows if the Estimated Project Value is Above GTE EM Approval Limit or Below GTE EM Approval Limit.
    Estimated Project Value Preliminary Estimated Value for the Potential Project.
    Expected Project Award Date Date for when the project is expected to get awarded.
    Get % 'Get' is how likely GTE is to win the project (assuming it goes ahead).
    Go % 'Go' is the likelihood the project is awarded (mostly a function of the client).
    Go-Get % Used in resource reports.
    GP%
    GTE Engineering Manager Holds the name of the current GTE Engineering Manager.
    GTE PM GTE project manager assigned to the project.
    GTE Proposal Manager Holds the name of the GTE proposal manager.
    GTE Technical Lead Holds the name of the GTE technical lead assigned to the project.
    ID Helper Helper column to make the project ID accessible to workflows. Obtained from the Sheet Summary field. Not to be modified manually.
    Initial Project Estimated Value ($) The estimated value of the project during it's potential phase in $AUD.
    Modified Date System generated column. Displays the date and time for when the row entry was last modified.
    Net Promotor Score
    New Value Banding Holds a value of either Extremely High, Very High, High, Medium or Low to provide a guide for the assessment and management of the project value. Details of which can be found in the Value Banding Overview.
    Notes Project specific notes can be added here.
    Numberic Risk Holds the Risk as a numerical value
    Portal - Closeout Holds "Jobchain" reference and link for the portal once the closeout provisioning has been completed. Populated by API, no formula.
    Portal - Proposal Provisioning Holds "Jobchain" reference and link for the portal once the proposal provisioning has been completed. Populated by API, no formula.
    Portal - Project Provisioning Holds "Jobchain" reference and link for the portal once the project provisioning has been completed. Populated by API, no formula.
    Portal - Proposal Submission Holds "Jobchain" reference and link for the portal once the project proposal submission has been completed. Populated by API, no formula.
    Potential Recommendation Approval Potential recommendation approval status: Submitted, Approved or Declined.
    Potential Recommendation Approval 1-Pass For projects classified as HIGH risk 2 directors are required to approve the bid/decline recommendation. This column stores the decision of the 1st director: Submitted, Approved or Declined.
    Potential Recommendation Approval 2-Pass For projects classified as HIGH risk 2 directors are required to approve the bid/decline recommendation. This column stores the decision of the 2nd director: Submitted, Approved or Declined.
    Potential Recommendation Approval 2IDR For projects classified as HIGH risk 2 directors are required to approve the bid/decline recommendation. This column stores the approval status based upon the decision made by both directors: Submitted, Approved or Declined.
    Potential Status Holds the current status of a potential project. Status can be: New, Assessing, Preparing, Submitted/Pending, Declined, Won, Lost or Cancelled.
    Predicted Profit % The percentage predicted profit.
    Project Awarded Date Date for when the projected is awarded.
    Project Closed Out Date Date for when the project has been closed out.
    Project Completed Date Date when the project is completed.
    Project Expected Start Date Date for when the project is expected to start after project has been awarded.
    Project Expected End Date Date for when the project is expected to be completed by.
    Project ID Unique 4-digit identifier for each project.
    Project Lost/Cancelled Details Project Manager to provide details regarding the cancellation or reasoning for the loss of a project.
    Project Name Name of the project.
    Project On-Hold? Checkbox for when the project goes on hold.
    Project Status Holds the current status of the project after it's potential status is Won. Project status is either Active or Complete.
    Proposal Approval Proposal Approval Status: Submitted, Approved or Declined.
    Proposal Approval 1-Pass For projects classified as HIGH risk 2 directors are required to approve the proposal. This column stores the decision made by the 1st director: Submitted, Approved or Declined.
    Proposal Approval 2-Pass For projects classified as HIGH risk 2 directors are required to approve the proposal. This column stores the decision made by the 2nd director.: Submitted, Approved or Declined.
    Proposal Approval 2DIR For projects classified as HIGH risk 2 directors are required to approve the proposal. This column stores the approval status based upon the decision made by both directors: Submitted, Approved or Declined.
    Proposal Approval Review Flag Checkbox to indicate whether the Proosal Manager has decided to Review the Declined Proposal for resubmission. Workflow (3.5) checks this and Approval Declined in Workflow (3.2A-D) clears this.
    Proposal Due Date Date for when the proposal is due by.
    Proposal Manager Bid/Decline Recommendation The recommended action to Bid or Decline set by the proposal manager.
    Proposal Outcome The outcome of the proposal: Won, Cancelled or Lost.
    Proposal Preparation Budget [hrs] The estimated budget hours assigned for project proposal preperation.
    Proposal Submitted Date Holds the date of proposal submission.
    Proposal Total Budget [$] vs GTE PM Approval Limit Shows if the Proposed Total Budget is Above GTE EM Approval Limit or Below GTE EM Approval Limit
    Recommendation Declined Approval request has been declined by the Engineering Manager or an Executive Director.
    Relevant Value for Banding Assessment Value used to calculate the Value Banding. Holds either the Total Awarded Budget [$] when the Stage is "Project" or the Estimated Project Value.
    Review Declined Proposal When a proposal is disapproved/declined the Proposal Manager has the option to either review and resubmit or cancel the proposal. This column stores that decision made by the Proposal Manager.
    Risk Provides the project risk classification which is determined through an assessment of the project value and complexity. Details of which can be found in the Classification Overview.
    Risk and Bid/Decline Verification Indicates wtherh whether the Bid or Decline recommendation is verified for the process to continue: Missing Bid/Decline Recommendation, Missing Risk Approver, Missing Commercial Risk Approver, Verified Bid, Verified Decline or Unverified.
    Risk Approval This determines the approval required for a given risk category.
    • Low Risk - Approval from Engineering Manager Req.
    • Moderate Risk - Approval of 1 Director Req.
    • High Risk - Approval of 2 Directors Req.
    • Extreme Risk - Approval from Board Req.
    Risk - Capability & Experience Holds a risk score of either:
    • 1 (Low) - High expertise/capability for the work
    • 2 (Med) Adequately experienced and capable
    • 3 (High) Marginally capable or no relevant experience (and/or 3rd party support required)
    Risk - Commercial Holds a risk score of either:
    • 1 (Low) - Contract in place
    • 2 (Med) - Vendor Number only
    • 3 (High) - No Commercial Agreement or GTE to sub-contract any portion of works
    Risk - Commercial Approve/Decline Indicates whether the Risk - Commercial approval has been: Submitted, Approved or Declined.
    Risk - Commercial Check Indicates whether aRisk - Commercial approval needs to occur: Requires Approval or Not Required..
    Risk - Schedule & Resource Availability Holds a risk score of either:
    • 1 (Low) - Comfortable timeline and preferred resources available to achieve it
    • 2 (Med) - Schedule achievable and resources available or obtainable to achieve it
    • 3 (High) - Unrealistic or unachievable schedule and/or desired resources not available
    Risk - Strategic Holds a risk score of either:
    • 1 (Low) - Directly relates to key strategic objectives or initiatives
    • 2 (Med) - No definitive alignment to strategic objectives, but not contravening operational intent
    • 3 (High) - Misalignment with strategic objectives or operational intents
    Risk - Technical & Reputational Holds a risk score of either:
    • 1 (Low) - Problem and solution are understood, and the solution is easily achievable
    • 2 (Med) - Problem is unclear, and solution is unclear or difficult to achieve. Reputational risk of failure is manageable
    • 3 (High) - Both problem and solution are difficult to define or understand, and solution is difficult to achieve. GTEs reputation is at risk in the event of failure
    Risk Banding @Award Set by automation as a snapshot in time.
    Risk Banding @Potential Set by automation as a snapshot in time.
    Risk Banding Holds a value of either High, Medium or Low to provide a guide for the assessment and management of the project risks.
    Risk Score Holds a risk score which is calculated as the sum of the Weighted Score - Capability & Experience, Weighted Score - Commercial, Weighted Score - Schedule & Resource Availability, Weighted Score - Strategic and Weighted Score - Technical & Reputational.
    Sales Source Sale source: Repeat Work, Referral, Website, Tender Portal, Cold Call, GTE BD, LinkedIn or Other.
    Sharepoint Site Required Indicates whether a Sharepoint Site is required to be setup by IT for the project after a successful quote.
    Slip Stream Project Indicates whether the project is a Slip Stream Project. For Slip Stream Projects, much of the process workflows are bypassed. This is auto-filled by the 'New Project Form'.
    Stage Holds the projects current stage: Closed, Potential, Proposed or Project.
    Subcontracting Approval Indicates whether the subcontractor's approval has been: Submitted, Approved or Declined.
    Subcontracting Check Indicates whether the a subcontractor's approval needs to occur: Requires Approval or Not Required.
    Subcontracting Services Indicates whether the a subcontractor services are required. Yes or No.
    Subcontracting Verification Indicates whether the subcontractor's approval is verified for the process to continue: Verified Approved, Verified Decline or Unverified.
    Total Approved Budget [$] Combined sum of awarded and variation budget
    Total Approved Budget [hrs] Combined sum of awarded and variation budget hours
    Total Awarded Budget [$] Project awarded and client's PO may differ (eg. May exclude some scope)
    Total Awarded Budget [hrs] Budget issued after project has been awarded by the client
    Total Change Budget [$] Any approved variations contribute to this
    Total Change Budget [hrs] Any approved variations contribute to this
    Total Requested Budget [$] Any variation requests not yet approved contribute to this
    Total Requested Budget [hrs] Any variation requests not yet approve contribute to this
    Total Submitted (Proposed) Budget [$] Project value derived from preparing the proposal
    Total Submitted (Proposed) Budget [hrs] Project value derived from preparing the proposal
    Type The type of project: Lump Sum, Rates or Lump Sum + Rates.
    Value Banding @Potential Set by automation as a snapshot in time.
    Weighted Score - Capability & Experience Holds the weighted score for Capability & Experience and used for calculation of the Risk Score.
    Weighted Score - Commercial Holds the weighted score for Commercial and used for calculation of the Risk Score.
    Weighted Score - Schedule & Resource Availability Holds the weighted score for Schedule & Resource Availability and used for calculation of the Risk Score.
    Weighted Score - Strategic Holds the weighted score for Strategic and used for calculation of the Risk Score.
    Weighted Score - Technical & Reputational Holds the weighted score for Technical & Reputational and used for calculation of the Risk Score.
    Workflow Log Logs the last workflow that was successfully triggered by automation.

    Formula

    =IF(
    OR(
        [Risk Score]@row < 1, 
        [Risk Score]@row > 3
    ),
    "ERROR: An out of bounds Risk Score has been calculated. Please check that the risks are all correctly set.", 
    IF(
        [Risk Score]@row < 1.4, 
        1, 
        IF(
            [Risk Score]@row < 1.8, 
            2, 
            IF(
                [Risk Score]@row < 2.2, 
                3, 
                IF(
                    [Risk Score]@row < 2.6, 
                    4, 
                    5
                )
            )
        )
    )
)

    GP %#

    Formula

    =IFERROR(
    (
        INDEX(
            {GTE Project Actuals - Spend to Date}, 
            MATCH(
                [Project ID]@row, 
                {GTE Project Actuals - Project ID}, 
                0
            )
        ) 
        - 
        INDEX(
            {GTE Project Costs - Costs to Date}, 
            MATCH(
                [Project ID]@row, 
                {GTE Project Costs - Project ID}, 
                0
            )
        )
    ) 
    / 
    (
        INDEX(
            {GTE Project Actuals - Spend to Date}, 
            MATCH(
                [Project ID]@row, 
                {GTE Project Actuals - Project ID}, 
                0
            )
        )
    ), 
    ""
)

    Numeric Risk#

    formula

    =IFERROR(
    VALUE(
        MID(
            Risk@row, 
            FIND(
                "(", 
                Risk@row
            ) + 1, 
            LEN(
                Risk@row
            ) - 
            FIND(
                "(", 
                Risk@row
            ) - 1
        )
    ), 
    ""
)

    Formula

    =IF(

    ),

                "Declined"
            )
        )
    )
)

    Potential Status#

    Formula

    =IF(

        [Proposal Outcome]@row <> "", 
        [Proposal Outcome]@row, 
        IF(
            [Potential Recommendation Approval]@row = "Approved", 
            IF(
                [Proposal Manager Bid/Decline Recommendation]@row = "Decline", 
                "Declined", 
                IF(
                    [Proposal Submitted Date]@row <> "", 
                    "Submitted/Pending", "Preparing"
                )
            ), 
            IF(
                [Potential Recommendation Approval]@row = "Submitted", 
                "Assessing", 
                "New"
            )
        )
    )
)

    Project Status#
