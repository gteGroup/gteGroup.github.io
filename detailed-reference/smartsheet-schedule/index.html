    Title Description Use Type Phase (Formula)
    % Complete Summarizes up through the WBS the % Complete which is determined at lower activity levels (as comp%). Summation/Propagation % Complete Actual comp%_all
    Activity The work item or summary header. Project Manager (Setup/Maintain) Metadata All
    Activity ID System autonumber column. System (Background) System All
    Actual [hrs] Summarizes up through the WBS the Actual Cost which is entered at the Cost Unit level (captured in the Cost System and input to the Project Schedule as actual_hours_recorded. Summation/Propagation Actual Cost Actual actual_hours_all
    Actual Finish Date Represents the date of completion of an activity, where this has occurred, and is equal to the date_finish_planned at that point. Project Team (Frontline Input) Date/Logic Actual
    Actual Recorded [hrs] Actual Cost in hours as captured in the Cost System and synced to the Project Schedule. Project Team (Frontline Input) Actual Cost Actual
    Actual Start Date Represents the date of commencement of an activity, where this has occurred, and is equal to the date_start_planned at that point. Project Team (Frontline Input) Date/Logic Actual
    Actual Variance to Baseline Finish Date Compares the Actual Finish Date (date_finish_actual) with the Baseline Finish Date (date_finish_baseline). Negative variance indicates a slip against the baseline. Calculation Metric Actual vs Baseline var_date_finish_actual_to_baseline
    Actual Variance to Baseline Start Date Compares the Actual Start Date (date_start_actual) with the Baseline Start Date (date_start_baseline). Negative variance indicates a slip against the baseline. Calculation Metric Actual vs Baseline var_date_start_actual_to_baseline
    AvazaSection System column for use with the API integration with Avaza. Calculation System All api_avaza_section
    Baseline Finish Date The finish date for the activity as agreed with the client either explicitly in the Baseline, or implicitly to support explicit commitments in the Baseline. Project Controls (Controlled) Date/Logic Baseline
    Baseline Planned % Complete Determines Schedule % Complete (or "Planned %") at all levels up through the WBS using the Planned Values (calculated at the lowest possibly determinable level as pv_budget_level and summarized upwards as pv_all) and summarized Budgets (budget_hours_all). Note that an activity and summary that does not have a budget (budget_hours_all), ie either directly assigned or from it's children, will not have Planned Value. As these may still be considered progressable, the baseline_schedule_%_complete is used as a fallback planned % for those activities or summaries, however the percentage does not impact higher level summaries. Summation/Propagation % Complete Baseline comp%_baseline_planned_all
    Baseline Schedule % Complete Determines the Schedule % Complete for an activity (or summary) at the data date (today) as a linearization of progress from the Baseline Start Date to the Baseline Finish Dates, factoring in non-working days in the project calendar. Note that the values yielded for summaries in the WBS are not accurate portrayals of Planned Value as these linearize at those summary levels only. Accurate "Planned %" is given by comp%_baseline_planned_all. Calculation % Complete Baseline comp%_schedule_planned_all
    Baseline Start Date The start date for the activity as agreed with the client either explicitly in the Baseline, or implicitly to support explicit commitments in the Baseline. Project Controls (Controlled) Date/Logic Baseline
    Budget [hrs] Summarizes up through the WBS the Budgeted Cost, in hours, which is entered at various budgeted levels (captured in the Baseline and input to the schedule as budget_hours_assigned. Summation/Propagation Budget Baseline budget_hours_all
    Budget Assigned [hrs] Budgeted Cost, in hours, as assigned to relevant individual activities. The sum of the assigned budgets equals the total project Budgeted Cost, and is summarised up through the WBS as budget_hours_all. Project Controls (Controlled) Budget Baseline
    Budget Assigned + Requested [hrs] Represents the approved Budgeted Cost, in hours, for the activity budget_hours_assigned plus any additional hours that have been requested formally but not yet been approved budget_hours_requested. Calculation Budget Planned budget_hours_assigned_and_requested
    Budget Issues Used to assist in assigning Budget using the budget_hours_assigned column, providing indication where overlapping budgets have been assigned, ie where assignments have been made on ancestors to an activity that has also been directly assigned a budget. Issues in this column will be represented by a symbol in the flag column if the Sheet Summary setting setting_show_budget_issues is checked. System (Display&Filter) Budget All budget_issues
    Budget Requested [hrs] Budgeted additional (unapproved and so not included in the Baseline) Cost, in hours, as assigned to relevant individual activities. Represents the sum of all additional hours requested for the activity (through formal change management with the client) that has not yet been approved and added to the (Baseline) Budget (budget_hours_assigned) for the activity. Project Controls (Controlled) Budget Planned
    Calculated % Complete Calculates the % Complete to use for the activity based on the comp%_evm_method selected. Calculation % Complete Actual comp%
    Client Hidden? Checking this column will allow the row to be hidden when applying the Filter - Client Hidden filter. System (Display&Filter) Metadata All
    Control Conditional Formatting System column. System (Display&Filter) System All control_cond_formatting
    Cost Code Entry column to define a Cost Unit (by assignment of Cost Code) where costs will be captured in the Cost System. Alternatively, the Cost Code can be assigned automatically based on the Cost Unit designations made by checking the cost_unit_designation column. Automatic assignment is the recommended option for a number of reasons and manual Cost Code assignment should not be done without consulting Project Controls. Project Controls (Controlled) Cost Coding All
    Cost Unit Optional column to designate a Cost Unit without assigning a Cost Code manually. The GTE Project Portal can then be used to auto-assign the Cost Codes to the designated Cost Units. The assigned Cost Codes will be populated to the cost_code column. Project Controls (Controlled) Cost Coding All
    Cost Unit Activity Combines the cost_unit_effective and the activity into a string. System (Display&Filter) Cost Coding All cost_unit_description
    Cost Unit Issues Used to assist in designating Cost Units using the cost_unit_designation column, providing details on any designation issues such as overlapping Cost Unit designations or budgeted activities not covered by a Cost Unit designation. Issues in this column will be represented by a symbol in the flag column if the Sheet Summary setting setting_show_cost_unit_issues is checked. System (Display&Filter) Cost Coding All cost_unit_issues
    CPI Cost Performance Index (based on hours). Calculation Metric Actual vs Baseline cpi
    CV Cost Variance (in hours). Calculation Metric Actual vs Baseline cv
    CV% Cost Variance (in hours), expressed as a percentage of Planned Value (hours). Calculation Metric Actual vs Baseline cv_%
    Default FTC [hrs] Calculates a default value to be used if a Forecast Hours to Finish (ftc_hours_entry) is not entered and there is assigned budget for the activity. If there is no assigned budget, the sum of the child activities is used. The value calculated is generally the full assigned budget (budget_hours_assigned) as a conservative estimate, however for an EVM Method (comp%_evm_method) of "Linearised LoE" the actual hours to date (actual_hours_recorded) will be deducted as these activities are expected to meet budget. Calculation Forecast Cost Forecast (to Baseline) ftc_default
    Default FTC Including Change [hrs] Calculates a default value to be used if a Forecast Hours to Finish (ftc_hours_entry) is not entered and there is assigned budget and/or requested hours (through formal change management with the client) for the activity. If there is no assigned or requested hours for the activity, the sum of the child activities is used. The value calculated is generally the full assigned and requested budget (budget_hours_assigned_and_requested) as a conservative estimate, however for an EVM Method (comp%_evm_method) of "Linearised LoE" the actual hours to date (actual_hours_recorded) will be deducted. Calculation Forecast Cost Forecast (to Planned) ftc_default_with_change
    Descendant Cost Units System column. System (Background) System All cost_unit_descendants
    Duration The working days between, and including, the Start (date_start_planned) and Finish Dates (date_finish_planned) of the activity; where working days are as defined by the project calendar (Smartsheet setting). Project Manager (Setup/Maintain) Date/Logic Planned
    Effective Cost Unit Propagates a Cost Code entered at the Cost Unit level down to descendant levels in the WBS for explicitly clarifying where the costs should be captured in the Cost System. System (Background) Cost Coding All cost_unit_effective
    Expected Budget [hrs] Summarizes hours up through the WBS as for budget_hours_all but includes additional hours that may have been requested of the client through formal change management, but not yet approved. The additional hours are input at the budgeted level as budget_hours_requested. Summation/Propagation Budget Planned budget_hours_with_change_all

    EV [hrs] Summarizes up through the WBS the Earned Value calculated at the lowest possibly determinable level (as ev_budget_level). The lowest possibly determinable level is where both % Complete and Budget Cost, in hours, exist. Summation/Propagation Earned Value Actual vs Baseline ev_all
    EV at Budget Level [hrs] Earned Value (in hours) at the budgeted activity level. Where an activity is budgeted, this is the Earned Value (in hours) for that activity as the budget multiplied by the physical % complete comp%. The duration of the activity is determined from the Baseline Start and Finish dates (as (baseline_schedule_%_complete), using date_start_baseline and date_start_baseline), and the budget used only includes the approved (Baseline) budget (budget_hours_assigned). Calculation Earned Value Actual vs Baseline ev_budget_level
    EVM Method The default method for determing an activity's % Complete (comp%) is using the manual entry column (comp%_manual). This column is used to select an alternate method such as "Linearised LoE", or "Gated". Project Manager (Setup/Maintain) % Complete All
    FAC [hrs] Forecast at Complete (in hours). This is simply the hours incurred to date (actual_hours_all) plus the forecast hours to complete (ftc_hours_all). Summation/Propagation Forecast Cost Forecast (to Baseline) fac_hours_all
    FAC Including Change [hrs] Forecast at Complete (in hours), factoring expected changes to the budget. This is simply the hours incurred to date (actual_hours_all) plus the forecast hours to complete, where in this instance this includes change (ftc_default_with_change). Summation/Propagation Forecast Cost Forecast (to Planned) fac_with_change_hours
    Filter - Client View System column. System (Display&Filter) System All filter_client_view
    Finish Date The Finish Date for the activity as most recently planned. This may differ from the Baseline equivalent (date_finish_baseline) and may or may not reflect field feedback (date_finish_fcast) depending on whether the Project Manager intends to hold or measure the project team to earlier plans. When an activity is complete, the Finish Date should equal the Actual Finish Date (date_finish_actual). Project Manager (Setup/Maintain) Date/Logic Planned
    Flag A cross (red) indicates an issue on the row. An exclamation (yellow) indicates an issue on a descendant row. The issues can be to do with Budgeting, Cost Coding, or Resourcing and the flags responsiveness to each is dependent on the selected options in the relevant 'Sheet Summary' settings. The specifics of the issues are detailed in the budget_issues, cost_unit_issues and/or resourcing_issues columns as relevant. System (Display&Filter) System All flag
    Forecast Finish Date The date, forecast by the relevant team member, when it is expected that the activity will be completed. This date informs the Project Manager's update of the (Planned) Start and Finish dates (date_start_planned and date_finish_planned), although the Project Manager will not necessarily change these dates based on the feedback. Project Team (Frontline Input) Date/Logic Forecast (to Baseline)
    Forecast Hours to Finish The hours, forecast by the relevant team member, required to finish the activity as considered on the basis of the work completed at the date of entry. The forecast should be reflective of the remaining work, and should not factor budget and/or hours burned to date. Project Team (Frontline Input) Forecast Cost Forecast (to Baseline)
    FTC [hrs] Forecast to Complete (in hours). Where a field forecast is applicable for an activity, generally where that activity is a Cost Unit, this value will be the field forecast (provided as ftc_hours_entry) if this has been provided. Where it has not, a default value will be used (ftc_default). Summation/Propagation Forecast Cost Forecast (to Baseline) ftc_hours_all
    FTC Including Change [hrs] Forecast to Complete (in hours), factoring expected changes to the budget. Where a field forecast is applicable for an activity, generally where that activity is a Cost Unit, this value will be the field forecast (provided as ftc_hours_entry) if this has been provided. Where it has not, a default value will be used where in this instance this includes change (ftc_default_with_change). Summation/Propagation Forecast Cost Forecast (to Planned) ftc_with_change_hours
    FUTURE - Parent for Apportioned Effort Future - disregard. FUTURE % Complete Actual
    FUTURE Deliverable Type Future - disregard. FUTURE Metadata All
    FUTURE Discipline Future - disregard. FUTURE Metadata All

    Information Free text entry to provide any comments against the activity that should be more visible than row comments. In some instances, this column may be populated from the original estimate and provide detail on the basis of the estimate for example. Project Manager (Setup/Maintain) Metadata All
    Manual % Complete Where "Manual % Complete" is selected as the earned value measurement method (comp%_evm_method), or where no selection is made, the physical % Complete of relevant activities should be entered in this column, general by project team members responsible for the work. Project Team (Frontline Input) % Complete Actual
    Modified By System column. System (Background) System All
    Non-Standard Capacity Standard resource capacity is 8hrs/day. Set this value where the activity is, for example, performed on site and involves 12hr days not standard 8hr office days. In this example case, type '12'. Project Manager (Setup/Maintain) Resourcing All
    Planned % Complete Determines Planned % Complete at all levels up through the WBS using the expected Planned Values, ie budgets including additional requests (calculated at the lowest possibly determinable level as pv_with_change_budget_level and summarized upwards as pv_with_change_all); and summarized Budgets (budget_hours_with_change_all). Note that an activity and summary that does not have a budget (budget_hours_with_change_all), ie either directly assigned or from it's children, will not have Planned Value. As these may still be considered progressable, the schedule_%_complete is used as a fallback planned % for those activities or summaries, however the percentage does not impact higher level summaries. Summation/Propagation % Complete Planned comp%_with_change_all
    Planned Variance to Baseline Finish Date Compares the (Planned) Finish Date (date_finish_planned) with the Baseline Finish Date (date_finish_baseline). Negative variance indicates a slip against the baseline. Calculation Metric Planned vs Baseline var_date_finish_planned_to_baseline
    Planned Variance to Baseline Start Date Compares the (Planned) Start Date (date_start_planned) with the Baseline Start Date (date_start_baseline). Negative variance indicates a slip against the baseline. Calculation Metric Planned vs Baseline var_date_start_planned_to_baseline
    Predecessors Used to define relationships between activities as schedule logic. Project Manager (Setup/Maintain) Date/Logic All
    Project System mirror of Sheet Summary value. System (Background) System All project_name
    PV [hrs] Summarizes up through the WBS the Planned Value (in hours) calculated at the budgeted (or lowest possibly "determinable") level (as pv_budget_level). Summation/Propagation Planned Value Baseline pv_all
    PV at Budget Level [hrs] Planned Value (in hours) at the budgeted activity level. Where an activity is budgeted, this is the Planned Value (in hours) for that activity as the budget multiplied by the percentage of time through the duration of the activity. The duration of the activity is determined from the Baseline Start and Finish dates (as comp%_schedule_planned_all), using date_start_baseline and date_finish_baseline), and the budget used only includes the approved (Baseline) budget (budget_hours_assigned). Calculation Planned Value Baseline pv_budget_level
    PVx [hrs] Summarizes up through the WBS the Planned Value (in hours) calculated at the budgeted (or lowest possibly "determinable") level as pv_with_change_budget_level, where this includes additional requested budget as well as approved/assigned budget (budget_hours_assigned_and_requested). Summation/Propagation Planned Value Planned pv_with_change_all
    PVx at Budget Level [hrs] Planned Value Expected (in hours) at the budgeted activity level. Where an activity is budgeted, this is the Planned Value (in hours) for that activity as that budget multiplied by the percentage of time through the duration of the activity. "Expected" means that in this instance the duration of the activity is determined from the Start and Finish dates from the current plan (as schedule_%_complete), using date_start_planned and date_finish_planned), and the budget used is actually the approved/assigned budget plus any additional requested budget (budget_hours_assigned_and_requested). Calculation Planned Value Planned pv_with_change_budget_level
    Reimbursable Activity Propagates the assignment of reimbursable activities, entered as a higher level designation (using reimb_designation) down to descendant levels in the WBS. Project Controls (Controlled) System All is_reimb_activity
    Reimbursable? For projects that include both lump sum and reimbursable (rates based) components, checking this column will designate the reimbursable items in the schedule. The designation will apply to all descendants of any checked item as determined by is_reimb_activity. Project Controls (Controlled) Metadata All
    Resource Loading Calculates the % of a single resources capacity required to complete a budgeted activity, where this a linear distribution of the budget over the duration of the activity divided by the number of resources and the daily capacity in hours of those resources (assumed to be 8hrs unless otherwise set using non_standard_capacity. System (Background) Resourcing All resource_loading
    Resource(s) Used to assign one or more human resources to budgeted activities. The assigned resource(s) share the budget requirement equally, linearised over the activity duration, for the purpose of assessing resource loading. Project Manager (Setup/Maintain) Resourcing All
    Resourcing Issues Used to assist in designating human resource(s) using the resources column, providing details on any missing assignments or overloaded resources, and where an activity duration has not been properly established to allow the resource loading to be determined. Issues in this column will be represented by a symbol in the flag column if the Sheet Summary setting setting_show_resourcing_issues is checked. System (Display&Filter) Resourcing All resourcing_issues
    Schedule % Complete Calculation % Complete comp%_schedule
    Sheet System mirror of Sheet Summary value. System (Background) System All sheet_name
    SPI Schedule Performance Index (based on hours). Calculation Metric Actual vs Baseline spi
    Start Date The Start Date for the activity as most recently planned. When an activity is complete, the Finish Date should equal the Actual Finish Date (date_start_actual). Project Manager (Setup/Maintain) Date/Logic Planned
    Status System column. System (Display&Filter) System All status
    SV Schedule Variance (in hours). Calculation Metric Actual vs Baseline sv
    SV% Schedule Variance (in hours), expressed as percentage of Planned Value (hours). Calculation Metric Actual vs Baseline sv_%
    WBS System column. System (Background) System All wbs
    WBS Descendants System column. System (Background) System All wbs_descendants

    Formulas#

    actual_hours_all#

    Formula

    =IF(
    AND([Actual Recorded [hrs]]@row <> "", [Actual Recorded [hrs]]@row <> 0), 
    [Actual Recorded [hrs]]@row, 
    SUM( CHILDREN([Actual [hrs]]@row) )
)

    api_avaza_section#

    Formula

    =IF(
    [WBS Level]@row <= 2, Activity@row, 
    INDEX( ANCESTORS(Activity@row), 2 )
)

    budget_hours_all#

    Formula

    =IF(
    AND([Budget Assigned [hrs]]@row <> "", [Budget Assigned [hrs]]@row <> 0), 
    [Budget Assigned [hrs]]@row, 
    SUM(CHILDREN())
)

    budget_hours_assigned_and_requested#

    Formula

    =[Budget Assigned [hrs]]@row + [Budget Requested [hrs]]@row

    budget_hours_with_change_all#

    Formula

    =IF(
   AND(
       [Budget Assigned + Requested [hrs]]@row <> "", 
       [Budget Assigned + Requested [hrs]]@row <> 0
    ), 
    [Budget Assigned + Requested [hrs]]@row, 
    SUM(CHILDREN())
)

    budget_issues#

    Formula

    =IF(
    AND( [Budget Assigned [hrs]]@row <> 0, [Budget Assigned [hrs]]@row <> "" ),
    IF(
        SUM( CHILDREN([Budget Assigned [hrs]]@row) ) > 0,
        "Budget also assigned on children",
        ""
    ),
    ""
)

    comp%#

    Formula

    =IF(
    [EVM Method]@row = "Manual % Complete", 
    [Manual % Complete]@row, 
    IF(
        [EVM Method]@row = "Linearised LoE", 
        [Planned % Complete]@row, 
        IF(
            [EVM Method]@row = "Gated", 
            INDEX({GatedProgressSummary - % Complete}, MATCH([Gated Progress Reference]@row, {GatedProgressSummary - Reference}, 0)), 
            IF(
                [EVM Method]@row = "Apportioned Effort", 
                "NOT SUPPORTED", 
                [Manual % Complete]@row
            )
        )
    )
)

    comp%_all#

    Formula

    =IF(
    PARENT([Calculated % Complete]@row) > 0, 
    PARENT([Calculated % Complete]@row),
    IF(
        [EV [hrs]]@row <> 0, 
        [EV [hrs]]@row / [Budget [hrs]]@row, 
        IF(
            AND([Budget [hrs]]@row <> 0, [Budget [hrs]]@row <> ""), 
            IFERROR( AVG(CHILDREN([Calculated % Complete]@row) ), ""), 
            [Calculated % Complete]@row
        )
    )
)

    comp%_baseline_planned_all#

    Formula

    =IF(
    [Budget [hrs]]@row <> 0, 
    [PV [hrs]]@row / [Budget [hrs]]@row, 
    [Baseline Schedule % Complete]@row
)

    comp%_schedule#

    Formula

    =IF(
    AND([Start Date]@row <> "", [Finish Date]@row <> ""), 
    ROUND(
        IF(
            TODAY() < [Start Date]@row, 
            0, 
            IF(
                TODAY() >= [Finish Date]@row, 
                1, 
                NETWORKDAYS([Start Date]@row, TODAY()) 
                    / NETWORKDAYS([Start Date]@row, [Finish Date]@row)
            )
        ), 
        2
    ), 
    0
)

    comp%_schedule_eomX#

    Formula

    =IF(
    AND([Start Date]@row <> "", [Finish Date]@row <> ""), 
    ROUND(
        IF(
            [EOM+X]# < [Start Date]@row, 
            0, 
            IF(
                [EOM+X]# >= [Finish Date]@row, 
                1, 
                NETWORKDAYS([Start Date]@row, [EOM+X]#) / NETWORKDAYS([Start Date]@row, [Finish Date]@row)
            )
        ), 
        2
    ), 
    0
)

    comp%_schedule_planned_all#

    Formula

    =IF(
    AND([Baseline Start Date]@row <> "", [Baseline Finish Date]@row <> ""),
    ROUND(
        IF(
            TODAY() < [Baseline Start Date]@row,
            0,
            IF(
                TODAY() >= [Baseline Finish Date]@row,
                1,
                NETWORKDAYS(
                    [Baseline Start Date]@row,
                    TODAY()
                ) / NETWORKDAYS( [Baseline Start Date]@row, [Baseline Finish Date]@row )
            )
        ),
    2),
    0
)
